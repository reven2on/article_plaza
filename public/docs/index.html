Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://localhost

Authenticating requests

This API is not authenticated.

Article management

Display a listing of the articles.

Example request:
curl --request GET \
    --get "http://localhost/api/articles?searchTerm=test&filter[date]=2022-06-04%2C2022-06-05&filter[categories]=minus%2Cpluses&sort[trending]=2022-06-04%2C2022-06-05&sort[popularity]=rating" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"searchTerm\": \"odit\"
}"

$client = new \GuzzleHttp\Client();
$response = $client->get(
    'http://localhost/api/articles',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'query' => [
            'searchTerm'=> 'test',
            'filter[date]'=> '2022-06-04,2022-06-05',
            'filter[categories]'=> 'minus,pluses',
            'sort[trending]'=> '2022-06-04,2022-06-05',
            'sort[popularity]'=> 'rating',
        ],
        'json' => [
            'searchTerm' => 'odit',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

Example response (200, Success):


{
    "message": "The article has been loaded",
    "data": [
        {
            "id": "1",
            "title": "test title",
            "body": "test body",
            "creation_date": "2022-06-04 14:44:02",
            "views": 180,
            "rating": 4.1
        }
    ]
}

Example response (429, Too many requests):


{
    "message": "Too many attempts, try again later"
}

Request      

GET api/articles

Query Parameters

searchTerm  string optional  

the phrase that will be searched.

filter[date]  string optional  

filter articles that have been created within a date range.

filter[categories]  string optional  

filter articles by one or more categories (comma separated string).

sort[trending]  string optional  

sort trenging articles by amount of views, where the date of views can be filtered

sort[popularity]  string optional  

sort articles by popularity based on amount of views or precise article rating (rating or view)

Body Parameters

searchTerm  string

Register a rating for an article.

every IP address can rate an article just once every IP address can rate total number of 10 per day

Example request:
curl --request POST \
    "http://localhost/api/articles/1/rate" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"rate\": 4
}"

$client = new \GuzzleHttp\Client();
$response = $client->post(
    'http://localhost/api/articles/1/rate',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'rate' => 4,
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));
